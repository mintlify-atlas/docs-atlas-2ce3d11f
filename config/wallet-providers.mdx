# Wallet Providers

The CLI supports **8 different wallet providers** across EVM and Solana chains. Each provider offers unique features like embedded wallets, social login, and multi-chain support.

## Provider Selection

Specify your wallet provider with the `--wallet` flag:

```bash
# EVM with RainbowKit
npx create-nextjs-dapp my-dapp --chain evm --wallet rainbowkit

# Solana with Wallet Adapter
npx create-nextjs-dapp my-dapp --chain solana --wallet wallet-adapter

# Multi-chain with Privy
npx create-nextjs-dapp my-dapp --chain evm --wallet privy
```

## EVM Wallet Providers

### RainbowKit

<Card title="Official Docs" icon="book" href="https://rainbowkit.com/docs">
  RainbowKit Documentation
</Card>

**Best for**: Production EVM dApps that need excellent UX

**Key Features**:
- Beautiful, polished wallet connection modal
- Support for 100+ wallets
- Excellent TypeScript support
- Customizable themes (light/dark)
- Built on wagmi and viem
- WalletConnect integration

**Dependencies**:
```json
{
  "@rainbow-me/rainbowkit": "^2.2.10",
  "wagmi": "^2.19.5",
  "viem": "^2.44.4"
}
```

**Environment Variables**:
```bash
NEXT_PUBLIC_WALLETCONNECT_PROJECT_ID=your_project_id
```

**When to Use**:
- You need a production-ready wallet connection UI
- You want support for many wallets out of the box
- You're building an EVM-only application
- You don't need embedded wallets or social login

### ConnectKit

<Card title="Official Docs" icon="book" href="https://docs.family.co/connectkit">
  ConnectKit Documentation
</Card>

**Best for**: EVM dApps with custom branding needs

**Key Features**:
- Highly customizable UI components
- Smooth animations and transitions
- Built by Family (formerly known for ENS)
- WalletConnect integration
- Customizable color schemes

**Dependencies**:
```json
{
  "connectkit": "^1.8.2",
  "wagmi": "^2.19.5",
  "viem": "^2.44.4"
}
```

**Environment Variables**:
```bash
NEXT_PUBLIC_WALLETCONNECT_PROJECT_ID=your_project_id
```

**When to Use**:
- You need extensive UI customization
- You want fine-grained control over the connect flow
- You're building an EVM-only application
- Design consistency with your brand is critical

### GetPara (Capsule)

<Card title="Official Docs" icon="book" href="https://docs.getpara.com">
  GetPara Documentation
</Card>

**Best for**: EVM dApps with embedded wallet requirements

**Key Features**:
- MPC (Multi-Party Computation) key management
- Embedded wallet creation
- Email-based authentication
- Non-custodial security model
- Programmable wallets

**Dependencies**:
```json
{
  "@getpara/react-sdk": "^2.0.0",
  "wagmi": "^2.19.5",
  "viem": "^2.44.4"
}
```

**Environment Variables**:
```bash
NEXT_PUBLIC_PARA_API_KEY=your_api_key
```

**When to Use**:
- You need embedded wallets with MPC security
- You want to abstract wallet complexity from users
- You're building consumer-facing applications
- You need EVM-only support

## Solana Wallet Providers

### Solana Wallet Adapter

<Card title="Official Docs" icon="book" href="https://github.com/solana-labs/wallet-adapter">
  Wallet Adapter Documentation
</Card>

**Best for**: Standard Solana applications

**Key Features**:
- Official Solana Labs implementation
- Support for all major Solana wallets
- Phantom, Solflare, Backpack, and more
- Standard wallet connection flow
- Auto-connect functionality
- Pre-built UI components

**Dependencies**:
```json
{
  "@solana/web3.js": "^1.98.0",
  "@solana/wallet-adapter-react": "^0.15.35",
  "@solana/wallet-adapter-react-ui": "^0.9.35",
  "@solana/wallet-adapter-wallets": "^0.19.32"
}
```

**No Environment Variables Required** (unless using custom RPC)

**When to Use**:
- You're building a Solana-only dApp
- You want the standard Solana wallet experience
- You don't need embedded wallets or social login
- You want maximum wallet compatibility

## Multi-Chain Wallet Providers

These providers support both EVM and Solana chains.

### Privy

<Card title="Official Docs" icon="book" href="https://docs.privy.io">
  Privy Documentation
</Card>

**Best for**: Apps that need email/social login with wallet support

**Key Features**:
- Email and social authentication (Google, Twitter, Discord, etc.)
- Embedded wallets for new users
- Progressive wallet onboarding
- Cross-app wallets
- EVM and Solana support
- Fiat on-ramps

**Dependencies (EVM)**:
```json
{
  "@privy-io/react-auth": "^2.4.1",
  "@privy-io/wagmi": "^1.0.2",
  "wagmi": "^2.19.5",
  "viem": "^2.44.4"
}
```

**Dependencies (Solana)**:
```json
{
  "@privy-io/react-auth": "^2.4.1",
  "@solana/web3.js": "^1.98.0"
}
```

**Environment Variables**:
```bash
NEXT_PUBLIC_PRIVY_APP_ID=your_app_id
```

**When to Use**:
- You want to onboard non-crypto users
- You need email/social login
- You want embedded wallets for new users
- You're building consumer-facing applications
- You need multi-chain support

**Dependencies (Solana)**:

**Environment Variables**:

**When to Use**:
- You want the official WalletConnect experience
- You need support for hundreds of wallets
- You're building a multi-chain application

## Decision Guide

Use this guide to choose the right wallet provider:

### For EVM Only:

<Steps>
  <Step title="Need just wallet connections?">
    Choose **RainbowKit** for the best UX and wide wallet support.
  </Step>
  
  <Step title="Need custom branding?">
    Choose **ConnectKit** for maximum UI customization.
  </Step>
  
  <Step title="Need embedded wallets?">
    Choose **GetPara** for MPC-based embedded wallets.
  </Step>
</Steps>

### For Solana Only:

<Steps>
  <Step title="Standard wallet connections?">
    Choose **Solana Wallet Adapter** - it's the recommended standard.
  </Step>
  
  <Step title="Need social login or embedded wallets?">
    Consider multi-chain providers like **Privy**, **Dynamic**, or **Thirdweb**.
  </Step>
</Steps>

### For Multi-Chain (EVM + Solana):

<Steps>
  <Step title="Need email/social login?">
    Choose **Privy** for the best progressive onboarding experience.
  </Step>
  
  <Step title="Need comprehensive platform features?">
    Choose **Thirdweb** for the most complete web3 development platform.
  </Step>
  
  <Step title="Need on-ramps and analytics?">
    Choose **Dynamic** for advanced features and user insights.
  </Step>
  
  <Step title="Need WalletConnect standard?">
    Choose **Reown (AppKit)** for the official WalletConnect implementation.
  </Step>
</Steps>

## Switching Providers

To switch wallet providers after project creation:

<Warning>
  Switching wallet providers requires replacing dependencies and updating provider configuration. It's often easier to create a new project.
</Warning>

<Steps>
  <Step title="Remove old dependencies">
    Uninstall the current wallet provider packages:
    
    ```bash
    npm uninstall @rainbow-me/rainbowkit  # or your current provider
    ```
  </Step>
  
  <Step title="Install new provider">
    Install the new wallet provider packages:
    
    ```bash
    npm install @privy-io/react-auth @privy-io/wagmi  # example
    ```
  </Step>
  
  <Step title="Update Providers.tsx">
    Replace the provider configuration in `components/Providers.tsx` with the new provider's setup.
  </Step>
  
  <Step title="Update environment variables">
    Add the required API keys and configuration for the new provider in `.env.local`.
  </Step>
  
  <Step title="Update components">
    Update your `Header.tsx` and other components that use wallet connection hooks.
  </Step>
</Steps>
