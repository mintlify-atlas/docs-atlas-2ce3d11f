## Overview

Dynamic is a comprehensive authentication and wallet platform that supports multi-chain (EVM + Solana), embedded wallets, social login, fiat on-ramps, and user analytics. It's designed for teams building **production-grade web3 applications** with advanced features.

**Best for**: Multi-chain applications that need comprehensive auth features, analytics, and embedded wallets with on-ramp capabilities.

## Key Features

- **Multi-Chain Support** - EVM and Solana chains in one integration
- **Embedded Wallets** - Auto-created wallets for new users
- **Social Login** - Email, Google, Twitter, Discord, and more
- **Fiat On-Ramps** - Built-in fiat-to-crypto conversion
- **User Analytics** - Detailed insights and user management dashboard
- **Account Abstraction** - Support for smart contract wallets
- **Customizable UI** - Extensive theming and branding options
- **wagmi Integration** - Full wagmi compatibility for EVM chains
- **One Widget** - Single component handles all authentication

## Configuration

Create a `.env.local` file:

```bash .env.local

  <Step title="Configure Settings">

    ```
  </Step>

### Provider Setup

The CLI generates a `Providers.tsx` component:

```tsx components/Providers.tsx
"use client";

import { ReactNode } from "react";
import { QueryClient, QueryClientProvider } from "@tanstack/react-query";
import { WagmiProvider, createConfig, http } from "wagmi";
import {
  DynamicContextProvider,
  DynamicWidget,
} from "@dynamic-labs/sdk-react-core";
import { EthereumWalletConnectors } from "@dynamic-labs/ethereum";
import { DynamicWagmiConnector } from "@dynamic-labs/wagmi-connector";
import { ThemeProvider } from "next-themes";
import { sepolia } from "wagmi/chains";

const queryClient = new QueryClient();

const environmentId = process.env.NEXT_PUBLIC_DYNAMIC_ENVIRONMENT_ID || "";

const config = createConfig({
  chains: [sepolia],
  transports: {
    [sepolia.id]: http(),
  },
  ssr: true,
});

const Providers = ({ children }: { children: ReactNode }) => {
  if (!environmentId) {
    console.warn(
      "Missing NEXT_PUBLIC_DYNAMIC_ENVIRONMENT_ID. Get one at https://app.dynamic.xyz/"
    );
  }

  return (
    <ThemeProvider attribute="class" defaultTheme="dark" enableSystem>
      <DynamicContextProvider
        settings={{
          environmentId,
          walletConnectors: [EthereumWalletConnectors],
        }}
      >
        <WagmiProvider config={config}>
          <QueryClientProvider client={queryClient}>
            <DynamicWagmiConnector>{children}</DynamicWagmiConnector>
          </QueryClientProvider>
        </WagmiProvider>
      </DynamicContextProvider>
    </ThemeProvider>
  );
};

export { Providers };
```

### Dynamic Hooks

Use Dynamic hooks to access user data and authentication state:

```tsx
import { useDynamicContext } from "@dynamic-labs/sdk-react-core";

const MyComponent = () => {
  const {
    user,                    // User data
    primaryWallet,          // Primary wallet
    wallets,                // All connected wallets
    setShowAuthFlow,        // Open auth modal
    handleLogOut,           // Log out user
    isAuthenticated,        // Auth state
    authToken,              // JWT token
  } = useDynamicContext();

  return (
    <div>
      {isAuthenticated ? (
        <div>
          <p>User ID: {user?.userId}</p>
          <p>Email: {user?.email}</p>
          <p>Wallet: {primaryWallet?.address}</p>
          <button onClick={handleLogOut}>Logout</button>
        </div>
      ) : (
        <button onClick={() => setShowAuthFlow(true)}>Login</button>
      )}
    </div>
  );
};
```

### Wagmi Integration

Use wagmi hooks with Dynamic:

```tsx
import { useAccount, useReadContract, useWriteContract } from "wagmi";
import { useDynamicContext } from "@dynamic-labs/sdk-react-core";

const ContractInteraction = () => {
  const { isAuthenticated } = useDynamicContext();
  const { address } = useAccount();
  
  const { data: greeting } = useReadContract({
    address: process.env.NEXT_PUBLIC_CONTRACT_ADDRESS as `0x${string}`,
    abi,
    functionName: "greeting",
  });

  const { writeContract, isPending } = useWriteContract();

  const handleUpdate = () => {
    writeContract({
      address: process.env.NEXT_PUBLIC_CONTRACT_ADDRESS as `0x${string}`,
      abi,
      functionName: "setGreeting",
      args: ["Hello from Dynamic!"],
    });
  };

  if (!isAuthenticated) {
    return <p>Please connect your wallet</p>;
  }

  return (
    <div>
      <p>Address: {address}</p>
      <p>Greeting: {greeting as string}</p>
      <button onClick={handleUpdate} disabled={isPending}>
        {isPending ? "Updating..." : "Update Greeting"}
      </button>
    </div>
  );
};
```

### User Information

Access comprehensive user data:

```tsx
import { useDynamicContext } from "@dynamic-labs/sdk-react-core";

const UserProfile = () => {
  const { user, primaryWallet, wallets } = useDynamicContext();

  return (
    <div>
      {/* User Info */}
      <p>User ID: {user?.userId}</p>
      <p>Email: {user?.email}</p>
      <p>Created: {user?.createdAt}</p>
      
      {/* Primary Wallet */}
      <p>Primary: {primaryWallet?.address}</p>
      <p>Chain: {primaryWallet?.chain}</p>
      
      {/* All Wallets */}
      {wallets.map(wallet => (
        <div key={wallet.id}>
          <p>{wallet.address} ({wallet.chain})</p>
        </div>
      ))}
    </div>
  );
};
```

### Multi-Chain Configuration

Support multiple EVM chains:

```tsx
import { mainnet, polygon, optimism, arbitrum, sepolia } from "wagmi/chains";

const config = createConfig({
  chains: [mainnet, polygon, optimism, arbitrum, sepolia],
  transports: {
    [mainnet.id]: http(),
    [polygon.id]: http(),
    [optimism.id]: http(),
    [arbitrum.id]: http(),
    [sepolia.id]: http(),
  },
  ssr: true,
});

<DynamicContextProvider
  settings={{
    environmentId,
    walletConnectors: [EthereumWalletConnectors],
    overrides: {
      evmNetworks: [
        { chainId: mainnet.id },
        { chainId: polygon.id },
        { chainId: optimism.id },
        { chainId: arbitrum.id },
        { chainId: sepolia.id },
      ],
    },
  }}
>
  {children}
</DynamicContextProvider>
```

## Troubleshooting

### Environment ID Missing

**Issue**: Console warning about missing Environment ID

**Solution**:
1. Visit [Dynamic Dashboard](https://app.dynamic.xyz)
2. Create a project and get your Environment ID
3. Add to `.env.local`:
   ```bash
   NEXT_PUBLIC_DYNAMIC_ENVIRONMENT_ID=abc123...
   ```
4. Restart dev server

### Widget Not Appearing

**Issue**: `DynamicWidget` doesn't render

**Solution**:
1. Verify `DynamicContextProvider` wraps your app
2. Check Environment ID is set
3. Ensure component is client-side (`"use client"`)
4. Check browser console for errors

### Wallet Connection Fails

**Issue**: Wallet connection doesn't work

**Solution**:
1. Verify the wallet connector is included:
   ```tsx
   walletConnectors: [EthereumWalletConnectors]
   ```
2. Check that the chain is supported in your Dynamic Dashboard
3. Verify wallet extension is installed and unlocked

### Wagmi Hooks Not Working

**Issue**: wagmi hooks return undefined

**Solution**: Ensure proper provider order:

```tsx
<DynamicContextProvider>
  <WagmiProvider>
    <QueryClientProvider>
      <DynamicWagmiConnector>
        {/* Components here */}
      </DynamicWagmiConnector>
    </QueryClientProvider>
  </WagmiProvider>
</DynamicContextProvider>
```

### Analytics Not Showing

**Issue**: User analytics not appearing in dashboard

**Solution**:
1. Wait a few minutes for data to sync
2. Verify Environment ID is correct
3. Check that events are being fired
4. Contact Dynamic support if issue persists
